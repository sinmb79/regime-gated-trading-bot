# 시스템 아키텍처 요약(보스용)

## 1) 전체 구성

- `data.py` : 시장 데이터 수집(현재는 Mock)
- `regime.py` : 시장 레짐 판별(확률형)
- `strategies/` : 전략 후보 생성기(그리드/추세/방어/Funding Arb/보조지표)
- `risk.py` : 리스크 게이트(손실한도, 신뢰도, 슬리피지, 쿨다운 등)
- `exchange.py` : 주문 인터페이스(현재 PaperExchange + CCXT Adapter)
- `execution.py` : 주문 실행·재시도·부분체결 집계·결과 기록
- `journal.py` : SQLite 복기 저장소
- `learning.py` : 성능 집계/전략별 제안 생성
- `pipeline.py` : 전체 순환 오케스트레이터
- `llm.py` : AI 요약기(현재 mock)
- `main.py` : CLI 진입점

## 2) 핵심 원칙

1. AI는 `설명 및 제안`만 수행
2. 실거래는 리스크 엔진과 규칙 엔진이 통제
3. 모든 결정은 기록으로 남겨서 복기 학습 가능
4. 전략은 전략 설정 파일 중심으로 켜고/끄고/비중 조정 가능
5. 주문 처리 단계에서 실패/부분체결을 분리해 실제 운영형 실패 모드를 반영

## 3) 기본 제어 흐름

1. 시세 수집
2. 레짐 판별
3. 전략 후보 생성
4. 후보 스코어링(EV/Confidence)
5. 리스크/쿨다운/EV 게이트 통과
6. 주문 실행(재시도/부분체결 집계)
7. 이벤트 저장
8. 일/주기별 전략 요약

## 4) 개선 포인트(운영 전환 전 필수)

- 실제 거래소 API 키 래핑(테스트넷/실거래 구분)
- 체결 실패/재시도 정책의 실제 지표화(부분체결률, 지연, 실패유형)
- 슬리피지 실측 및 주문 미체결 큐 관리
- 심볼별 포지션 분산 제약 추가
- 전략 파라미터 버전 관리
- 실시간 모니터링 UI(상태, 노출, 미체결, 경고)
